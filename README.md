# proposal-editor

专业型文档修改助手。当你对项目建议书、可行性研究报告这类专业文档做修改时，这个skill会自动介入，提供逻辑联动检查、多色高亮、数据校验等专业功能。

## 适用人群

- 经常需要修改项目型文档（建议书、可研报告、立项报告等）的用户
- 需要保持文档内部数据逻辑一致性的用户
- 希望修改内容有清晰高亮标记的用户

## 安装方法

1. 下载本仓库的 zip 包
2. 在 CodeBuddy/WorkBuddy 的专家中心添加新 skill
3. 导入 proposal-editor.zip

## 示例场景

**场景一：预算数据修改**

把预算从XX万改成YY万，skill会自动：
1. 识别相关联动项（收益测算、投资回收期等）
2. 用绿色高亮标记预算修改
3. 用蓝色高亮标记相关联的数据调整
4. 校验全文数字单位是否统一

**场景二：项目定位调整**

新增某种定位描述，skill会自动：
1. 识别相关联动项（背景描述、建设目标等）
2. 用黄色高亮标记新增内容
3. 检查全文定位表述是否统一

## 注意事项

1. 示例中使用"XX项目"、"YY万元"等抽象名称，不涉及真实项目信息
2. skill会优先检查逻辑联动，修改前会提示相关联的待调整项
3. 所有修改都会用对应颜色的高亮标记，方便审阅
